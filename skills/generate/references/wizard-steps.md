# ウィザードステップ詳細

各ステップの質問文、バリデーションルール、デフォルト値の定義。

## フロー概要

ウィザードは **2ステップ + 確認** のみで完了する。カスタマイズが必要な設定は、生成後に各adminコマンドで行う。

```
Step 1: 基本情報（必須）
Step 2: ストレージ選択 + 接続情報（必須）
確認:   デフォルト設定の説明 → 生成実行
```

---

## Step 1: 基本情報

### チーム名
- 質問: 「プラグインを使うチーム名を教えてください」
- 例: 「SaaS営業部」「プロダクト推進部」
- バリデーション: 空でないこと

### 事業名（商材・サービス名）
- 質問: 「対象の事業（商材・サービス名）を教えてください」
- 例: 「Zendesk」「Datadog」「Auth0」「Twilio/SendGrid」「AWS GameLift」
- 補足: SaaS商材に限らず、自社サービスやプラットフォーム等も対象
- バリデーション: 空でないこと

### コマンドプレフィクス
- 質問: 「コマンド名の先頭に付ける略称を決めてください（2-4文字の英小文字）」
- 自動提案ロジック:
  - 商材名の先頭2文字を小文字化（Zendesk → zd, Datadog → dd, Auth0 → au）
  - 複合名の場合は各語の頭文字（Twilio/SendGrid → tw）
  - 3文字以上が望ましい場合は先頭3文字（例: dat）
- バリデーション: 2-4文字、英小文字のみ、既存プラグインと重複しないこと
- デフォルト: 自動提案値

---

## Step 2: ストレージ選択

### ストレージ種別
- 質問: 「データの保存先を選んでください」
- 選択肢:
  - **Backlog Wiki** — チーム共有向け。Backlog MCPツールで操作
  - **Obsidian Vault** — 個人利用向け。Obsidian MCPツールで操作
- デフォルト: Backlog Wiki（チーム利用を想定）

### 接続情報（Backlog Wiki選択時）
- 質問: 「Backlogのプロジェクトキーを入力してください」
- 例: 「ZENDESK_PRJ」「TWILIO_SG_PRJ」
- 補足: 入力値がそのままプロジェクトキーとして使用される（自動プレフィクス付与なし）
- バリデーション: 英大文字・数字・アンダースコアのみ

### 接続情報（Obsidian Vault選択時）
- 質問: 「Obsidian Vault内のベースパスを教えてください（ルートからの相対パス）」
- 例: 「zendesk」「teams/zendesk」
- バリデーション: 有効なパス文字列

---

## 確認: デフォルト設定の説明

Step 2 完了後、以下のデフォルト設定を説明し、生成の確認を取る。

表示内容:
```
以下のデフォルト設定でプラグインを生成します:

- ナレッジカテゴリ: 製品・技術仕様 / 業務フロー・ガイドライン（必要に応じて追加可能）
- 営業フレームワーク: BANTCH（Budget, Authority, Need, Timeline, Competitor, Human resources）
- KPI売上内訳: 未設定（生成後にコマンドで設定可能）
- データソース: Slack, Google Calendar, Gmail, Google Drive（有効）
- 競合情報・料金体系: 未設定（生成後にコマンドで設定可能）
- Slackチャンネル: 未設定（生成後にコマンドで設定可能）

生成後、`/{prefix}-admin-setup` を実行するとセットアップガイドが表示されます。
よろしいですか？
```

## デフォルト値一覧

ウィザードで聞かなくなった設定項目のデフォルト値。生成後に各adminコマンドで変更可能。

| 設定項目 | デフォルト値 | 変更コマンド |
|---------|------------|------------|
| ナレッジカテゴリ | 製品・技術仕様, 業務フロー・ガイドライン | ナレッジカテゴリはストレージのディレクトリ構造で管理 |
| サブカテゴリ | なし | ストレージに直接ディレクトリ追加 |
| 営業フレームワーク | BANTCH | コンテキストページのフレームワークセクションを直接編集 |
| 競合情報 | 空（未設定） | `/{prefix}-admin-competitors` |
| 料金体系 | 空（未設定） | `/{prefix}-admin-pricing` |
| KPI売上内訳カテゴリ | 未設定（売上合計のみ） | `/{prefix}-admin-kpi-set` |
| Slackチャンネル | 空（未設定） | `/{prefix}-admin-slack` |
| Backlog監視プロジェクト | 空（未設定） | `/{prefix}-admin-backlog` |
| データソース | Slack, GCal, Gmail, GDrive有効 / Backlog Issues無効 | `.team-config.yml` を直接編集 |
| コマンド除外 | なし（全コマンド生成） | 不要コマンドのファイルを削除 |
