## 実装詳細

テンプレートファイル変更時にコミット前必須実行する10項目の整合性チェック定義。

### 10項目チェック

| チェック | 内容 |
|---------|------|
| 1 | コマンド→スキル呼び出しチェーン: 各コマンドが参照するセクション名がスキルに存在するか |
| 2 | 命名規則の一貫性: `{{product_prefix}}-deal` 統一（contextではない） |
| 3 | テンプレート変数の定義↔使用: `{{xxx}}` がconfig-schema/template-assembly/storage-adaptersで定義されているか |
| 4 | 出力パスマッピング: template-assembly.mdのマッピング表と実ファイルの一致 |
| 5 | config-schema ↔ wizard-steps: ウィザードで収集する情報とスキーマの対応 |
| 6 | plugin-json.template: 生成されるplugin.jsonのスキル名・コマンド名と実ファイルの一致 |
| 7 | forge自体のcommands/ ↔ skills/: forgeのコマンドとスキルの参照整合性 |
| 8 | 判断フロー: dealスキルの判断フローに記載されたコマンド名の存在確認 |
| 9 | 詳細リファレンス: スキル末尾のreferences一覧と実ファイルの一致 |
| 10 | ストレージアダプタ: `{{storage_*_cmd}}` 変数が両アダプタで定義されているか |

### 実行条件（CLAUDE.mdに定義）

| 変更ファイル | 必須チェック |
|------------|------------|
| `templates/` の `.template` ファイル1つでも変更 | 全10項目 |
| `storage-adapters/` を変更 | チェック3, 10最低限 |
| `skills/generate/references/` を変更 | チェック3, 4, 5最低限 |
| `skills/generate/SKILL.md` を変更 | チェック7最低限 |
| `README.md`, `CLAUDE.md`, `LICENSE` のみ | チェック不要 |

### 検証の原則

- **推測禁止**: ファイルを読んで事実ベースで判断する
- **全件確認**: 部分的な抽出チェックではなく全件確認
- **差分報告**: 問題がなくても確認内容を報告
