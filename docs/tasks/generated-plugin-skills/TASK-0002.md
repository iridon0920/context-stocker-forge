## 実装詳細

### サブコマンド実装

**setup**
- 未設定項目をチェックして設定ガイドを表示
- 設定コマンドの一覧と現在の設定状況を提示

**index**
- ストレージの全ページを取得・集計してINDEXを再構築
- `{{storage_index_rebuild_procedure}}` の手順に従う

**slack / backlog / competitors / pricing**
- 各設定ページをストレージから読み込み → 編集 → 保存
- 書き込み確認プロトコルを適用

**members**（`{{^excluded_admin_members}}` で制御）
- チームメンバー設定ページの管理
- logスキルのチームメンバー別サマリーと連動

**kpi-set / okr-set**
- KPI・OKR設定ページの管理
- `{{#kpi_revenue_categories}}` ループで売上内訳カテゴリを展開

**stale**
- `{{#stale_thresholds}}` 定義の閾値（warn_days / danger_days）でページ鮮度を確認
- 期限超過ページをリスト表示

**migrate**
- `templates/migrations/` のルールに従ってデータフォーマットを移行
- バッチ処理オプション（全件/カテゴリ別/N件）

### セッション初期化

3-Phase初期化でセッションキャッシュを構築:
- Phase 1: ストレージ接続確認
- Phase 2: 設定ページwikiIdを並列取得
- Phase 3: 設定ページ本文を並列取得・パース・キャッシュ
