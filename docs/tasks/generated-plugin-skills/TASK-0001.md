# TASK-0001: dealスキル（案件コンテキスト管理）

- **推定工数**: 5h
- **依存タスク**: storage-adapter-system TASK-0001 or TASK-0002

## 実装ファイル

- `templates/skills/deal/SKILL.md.template`

- `templates/skills/deal/references/similarity-check.md.template`

## 実装詳細

### コンテキスト復元

1. INDEXページからwikiId取得（最適化D: キャッシュ活用で `get_wiki_pages` スキップ）
2. 案件ページを読み込みセッションキャッシュに展開
3. 営業フレームワークフィールド（`{{#sales_framework_fields}}`）を表示

### コンテキスト保存

1. 既存ページの有無を確認
2. 営業フレームワークフィールドを収集・フォーマット
3. ストレージに保存（書き込み確認プロトコル）
4. INDEXを更新

### セッションキャッシュルール（最適化C）

| キャッシュ項目 | 取得タイミング | 無効化条件 |
|-------------|-------------|-----------|
| projectId | Phase 1 | セッション終了 |
| 設定ページwikiId | Phase 2 | セッション終了 |
| 設定ページ本文 | Phase 3 | 書き込み操作後 |

### INDEXフォーマット（最適化D）

INDEXページにwikiId列を追加し、読み込み時の `get_wiki_pages` 検索をスキップ:
```
| 顧客名 | 案件名 | ステータス | wikiId |
```

### 類似案件検索

`similarity-check.md` に基づき、同業種・類似課題の案件を推薦する。

### referenceファイル

- `context-format.md`: 案件コンテキストページのMarkdownフォーマット定義
- `index-format.md`: INDEXページフォーマット（wikiId列含む）
- `similarity-check.md`: 類似案件判定ルール
