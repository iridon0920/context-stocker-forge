## 実装詳細

### daily（デイリーログ記録）（最適化B）

3-Phase並列データ収集:
- **Phase 1**: セッション初期化（ストレージ接続・設定読み込み）
- **Phase 2**（並列）: Slackバッチ検索 + Google Calendar取得 + Backlog課題取得 + Gmail検索
- **Phase 3**: データ統合 → デイリーログ生成 → ストレージ保存

**Slackバッチ検索**（最適化E）: チャンネル別個別検索から `(in:#ch1 OR in:#ch2 OR ...)` バッチクエリに統合。

### weekly（週次レポート生成）

- 指定週のデイリーログを全件取得
- 重要度ティア（高/中/低）に分類して絞り込み
- 週次レポートフォーマット（weekly-report-format.md）に従って生成・保存

### report（階層型レポート集約）

- 週次レポートを集約して月次レポートを生成
- 月次レポートを集約して四半期レポート・年次レポートを生成
- フォールバックロジック: 集約元データが不足している場合はデイリーログから直接集約

### チームメンバー別サマリー

`{{^excluded_admin_members}}` 条件ブロックで制御。adminスキルで設定されたメンバー情報に基づいてメンバー別の活動サマリーを生成。

### referenceファイル

- `daily-log-format.md`: デイリーログのMarkdownフォーマット定義
- `weekly-report-format.md`: 週次レポートのMarkdownフォーマット定義
