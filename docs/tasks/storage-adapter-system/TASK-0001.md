## 実装詳細

### 変数契約（27変数）

このアダプタは `template-assembly.md` の「カテゴリ3: backlog-wiki選択時の値」に定義された27変数を全て提供する。

必須変数: `storage_name`（`Backlog Wiki`）, `storage_create_cmd`（`add_wiki`）, `storage_read_cmd`（`get_wiki`）, `storage_update_cmd`（`update_wiki`）, `storage_search_cmd`（`get_wiki_pages(keyword: ...)`）, `storage_session_init`（3-Phase手順）, 他22変数

### 3-Phase初期化（最適化A）

旧方式（全直列6〜8ターン）を3フェーズに短縮:
- **Phase 1**（直列・必須）: `get_project(projectKey: "{{project_key}}")` でprojectId取得
- **Phase 2**（並列）: `get_wiki_pages` でHome・各設定ページのwikiIdを一括取得
- **Phase 3**（並列）: Phase 2のwikiIdで `get_wiki` し本文取得・パース・キャッシュ

### wikiIdキャッシュ高速パス（最適化D）

`get_wiki_pages` による検索をスキップして直接 `get_wiki(wikiId: {既知のwikiId})` を呼び出す:
- セッション初期化Phase 2で取得済みページ（Home・設定ページ）
- INDEXページのwikiId列に記載あり（コンテキスト・ナレッジページ）
- 同一セッション内で取得済みのwikiId

### ページ操作

| 操作 | MCPツール | 備考 |
|------|---------|------|
| 作成 | `add_wiki(projectId, name, content)` | ページパスはnameで表現 |
| 読み込み | `get_wiki_pages` → `get_wiki` | 2段階（wikiId既知時は1段階） |
| 更新 | `update_wiki(wikiId, content)` | 更新前に既存内容を読み込み |
| 検索 | `get_wiki_pages(keyword)` | 結果をページ名でフィルタ |
| 一覧 | `get_wiki_pages(projectId)` | プレフィクスでフィルタ |

### ページパス命名規則

- コンテキスト: `案件/{顧客名}/{案件名}`

- INDEX: `INDEX/Home`, `INDEX/顧客・案件` 等
