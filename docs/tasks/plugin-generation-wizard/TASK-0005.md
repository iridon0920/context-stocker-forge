## 実装詳細

### ZIPパッケージングルール

バリデーション全PASS後、プラグインディレクトリの中身をZIPアーカイブとして出力する。

**正しいパッケージング方法（ラッパーディレクトリ禁止）**:
```bash
cd {output_dir}/{plugin_name}
zip -r ../{plugin_name}.plugin .claude-plugin/ skills/ commands/ .team-config.yml README.md
```

- ZIPのルート直下に `.claude-plugin/plugin.json` が存在する構造にすること
- `{plugin_name}/` のようなラッパーディレクトリで囲まない（コマンドが認識されない原因になる）
- 出力先はユーザーに確認（デフォルト: カレントディレクトリ）

### 生成後ガイダンス

プラグイン生成完了時に以下を案内:
1. `.plugin` ファイルのインストール方法
2. 初回セットアップコマンド（`/{pre}-admin setup`）の実行推奨
3. 設定ファイル（`.team-config.yml`）はプラグイン内に同梱済みであること（再生成時に自動で読み込まれる）

### plugin.json 生成ルール

`plugin.json` に含めるフィールドは `name`, `version`, `description`, `author`, `keywords` のみ。
`skills`, `commands`, `data_sources`, `format_version` 等の追加フィールドは禁止（コマンドが認識されない原因になる）。
