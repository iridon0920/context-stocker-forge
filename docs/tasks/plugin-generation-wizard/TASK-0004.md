## 実装ファイル

## 実装詳細

パッケージングの前に必ず実行する検証プロセス。1項目でもNGの場合はStep 4（テンプレート合成）に戻って修正・再チェック。

### チェック1: 未解決テンプレート変数の検出

- 全出力ファイルで `{{` パターンを検索
- `.team-config.yml` と `forge-consistency-check-prompt.md` は除外
- 0件 → PASS、1件以上 → NG

よくある未解決変数: `{{storage_project_key}}`, `{{storage_base_path}}`, `{{drive_folder_id}}`, `{{default_channels_list}}`

### チェック2: 商材名の一貫性

- `product_name` が正しく展開されているか
- 既知の他商材名（Twilio, SendGrid, Zendesk等）が混入していないか

### チェック3: コマンドプレフィクスの一貫性

- `commands/` 内の全ファイル名が `{product_prefix}-` で始まっているか
- ファイル内のコマンド参照も正しいプレフィクスか

### チェック4: スキル参照の整合性

- コマンドファイル内のスキル参照（plugin_name, スキル名）が正しいか
- `plugin.json` の `name` とスキルディレクトリ名が一致するか

### チェック5: ストレージ設定の整合性

- `project_key` / `base_path` の値が `.team-config.yml` と一致するか
- ストレージ操作セクション内の変数が全て解決済みか

### チェック6: ファイル構成の検証

- 必須ファイルが全て存在するか（`.claude-plugin/plugin.json`, `README.md`, 全スキルファイル等）
- excluded_commandsで除外したセクションが含まれていないか
- referenceファイルが全て存在するか
