## 実装詳細

### テンプレート変数記法（Mustache風）

| 記法 | 用途 |
|------|------|
| `{{variable}}` | 単純置換 |
| `{{#array}}...{{/array}}` | 配列ループ展開 |
| `{{#condition}}...{{/condition}}` | 条件ブロック（truthy） |
| `{{^condition}}...{{/condition}}` | 否定条件ブロック（falsy） |
| `{{storage_operations}}` | ストレージアダプタ全内容を差し込み |

### 変数カテゴリ（5種類）

1. **config直接マッピング**: `{{product_name}}`, `{{product_prefix}}`, `{{storage_project_key}}` 等
2. **派生値**: `{{plugin_name}}`, `{{skill_deal_name}}`, `{{skill_reference}}`, `{{default_channels_list}}`, `{{sales_framework_sections}}` 等 20+変数
3. **ストレージ個別変数**: ストレージ種別に応じて `{{storage_name}}`, `{{storage_create_cmd}}`, `{{storage_session_init}}` 等 27変数
4. **配列ループブロック**: `{{#knowledge_categories}}`, `{{#sales_framework_fields}}`, `{{#default_slack_channels}}` 等 20+ブロック
5. **条件ブロック**: `{{^excluded_admin_setup}}`, `{{#data_sources.slack.enabled}}`, `{{#drive_folder_id}}` 等

### 7ステップの処理フロー

1. **設定ファイル読み込み**: `.team-config.yml` をパース
2. **派生値の計算**: カテゴリ2の全変数をcontext辞書に格納
3. **ストレージ変数の定義**: ストレージ種別に応じてカテゴリ3の全変数を定義
4. **テンプレートファイルの処理**: 展開順序 → storage_operations差込 → ループ → 条件 → 単純置換
5. **コマンド除外の適用**: `excluded_*` フラグ変数を生成し条件ブロックで評価
6. **生成物チェック（必須）**: 6項目のバリデーション
7. **パッケージング**: ZIPアーカイブ生成
