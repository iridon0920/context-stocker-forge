## 実装詳細

### モード判定

| 引数 | モード |
|------|--------|
| 引数なし | 新規ウィザードモード（Step 1から開始） |
| 引数あり（プラグインパス） | 再生成モード（`.team-config.yml` 読込 → 変更確認 → 合成） |
| migrateコマンドから | マイグレーションモード |

### ウィザードフロー（新規）

**Step 1: 基本情報（必須）**
- チーム名: 空でないことを確認
- 事業名: 空でないことを確認
- コマンドプレフィクス: 商材名から自動提案（Zendesk→zd, Datadog→dd）/ 2-4文字英小文字 / 既存プラグインと重複不可

**Step 2: ストレージ選択（必須）**
- 種別: Backlog Wiki（チーム共有向け）/ Obsidian Vault（個人利用向け）
- 接続情報（Backlog): プロジェクトキー（例: ZENDESK_PRJ）/ 英大文字・数字・アンダースコアのみ
- 接続情報（Obsidian): ベースパス（例: zendesk）/ 有効なパス文字列

**Step 3: 営業フレームワーク選択（デフォルトあり）**
- BANTCH（デフォルト）/ BANT / MEDDIC / カスタム
- カスタム選択時はフィールド定義の入力

**Step 4: データソース選択（デフォルトあり）**
- Slack / Google Calendar / Gmail / Google Drive / Backlog Issues
- デフォルト: 全データソース有効

**Step 5: ナレッジカテゴリ選択（デフォルトあり）**
- 必須カテゴリ（削除不可）+ 任意追加カテゴリ
- デフォルトカテゴリあり、最低1件必須

**確認: 全設定一覧の表示**
- Step 1-5で収集した全設定を一覧表示
- 生成後の設定方法案内（`/{prefix}-admin setup`）
- ユーザーの承認を取得してから合成に進む

### 再生成モード

1. 対象プラグインから `.team-config.yml` を読み込み
2. 現在の設定内容をサマリー表示
3. 変更したい項目があるか確認
4. 変更があれば該当項目のみ再入力
5. `.team-config.yml` を更新保存して合成実行
