## 実装ファイル

- `skills/generate/references/config-schema.md` — 設定ファイルの完全スキーマ定義

## 実装詳細

### トップレベルフィールド

| フィールド | 型 | 必須 |
|-----------|-----|------|
| `format_version` | integer | Yes (現在: 1) |
| `product_name` | string | Yes |
| `product_prefix` | string | Yes (2-4文字英小文字) |
| `team_name` | string | Yes |
| `plugin_name` | string | Yes (自動生成) |
| `storage` | object | Yes |
| `knowledge_categories` | array | Yes |
| `sales_framework` | string | Yes |
| `data_sources` | object | Yes |
| `excluded_commands` | array | No |

### storage オブジェクト

- `type`: `backlog-wiki` または `obsidian-vault`
- `backlog_wiki.project_key`: Backlogプロジェクトキー
- `obsidian_vault.base_path`: Vault内ベースパス

### 組み込み営業フレームワーク

- **BANTCH**: Budget / Authority / Need / Timeline / Competitor / Human Resources
- **BANT**: BANTCHからCompetitor, Human Resourcesを除いた4項目
- **MEDDIC**: Metrics / Economic Buyer / Decision Criteria / Decision Process / Identify Pain / Champion

### knowledge_categories デフォルト

- 製品・技術仕様（required: true）
- 業務フロー・ガイドライン（required: true）

### data_sources

各データソース（Slack, Google Calendar, Gmail, Google Drive, Backlog Issues）に `enabled` フラグ。
Slackのみ `default_channels` 配列も定義（name, id, usage）。

### excluded_commands

除外可能なコマンド名リスト。統合コマンド（admin, doc, engdoc, log）はセクション単位除外、deal/knowledgeはファイル単位スキップ。
