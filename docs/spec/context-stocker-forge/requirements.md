## システム概要

### 推定されたシステム目的

context-stocker-forge は、B2B事業チーム向けコンテキスト管理プラグイン（context-stocker）を対話型ウィザードで自動生成する **メタプラグイン**（プラグインを生成するプラグイン）である。

Claude Code プラグインシステム上で動作し、Mustache風テンプレートエンジンと `.team-config.yml` 設定ファイルを組み合わせることで、各チーム専用のプラグインを生成する。

### 対象ユーザー

| ユーザー種別 | 説明 |
|------------|------|
| プラグイン管理者 | context-stocker-forge を使って自チーム用プラグインを生成・管理する担当者 |
| コントリビューター | context-stocker-forge 自体を開発・改善する開発者 |

## 推定されていない要件・確認事項

### ステークホルダー確認が必要な項目

1. **対応チーム規模**: 何名規模のチームを想定しているか（ウィザードのデフォルト値に影響）
2. **ストレージ追加優先度**: 次に追加すべきストレージアダプタ（Notion / GitHub Wiki 等）
3. **マイグレーション自動化**: バージョンアップ時の自動マイグレーション提供方針
4. **マーケットプレイス公開**: `marketplace.json` の位置付けと公開予定

### 不明確な要件

1. **同時実行**: 複数チームが同時に `/forge-generate` を実行した場合の排他制御（Claude Code 側の制御に依存）
2. **ロールバック**: 生成失敗時の部分生成物の扱い（現状は未定義）
3. **プラグインバージョン管理**: 生成済みプラグインのバージョン更新フロー
